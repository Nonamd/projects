Position: relative

When setting the position to relative, elements will still appear within the normal flow a page but will unlock the additional offset properties (top, right, bottom, left). You can use these values to nudge the element in different directions. Try adding a top value (or any offset property).

Position: absolute

When positioning an element as absolute, it can be tricky. It needs to also be paired with another positioning property, added to a parent element, to contain it to a particular spot.

Usually relative positioning is used because it doesn't affect other elements around it. However, the absolute values will be relative to the next parent element with relative, absolute or fixed positioning. If there is no such parent, it will default to the browser window.

<div class="container">
    <div class="box"></div>
</div>
Try removing position: relative; in the example below and see what happens.

Position: fixed

When positioning an element as fixed, it will always be positioned to the viewport. relative or absolute won't have any effect on this property. Also, it will always stay fixed on the page, even on page scroll.

<div class="container">
    <div class="box"></div>
</div>
